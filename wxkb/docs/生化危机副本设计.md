# 生化危机副本设计（研究所突破）

> 用途：作为后续 AI/开发实现该副本时的统一参考文档。  
> 目标：把“生化危机”做成一个 **现代 MUD 风格** 的手动探索副本，而不是另一个自动挂机页面。

---

## 1. 设计定位

### 1.1 副本在整款游戏中的职责

本游戏已经存在挂机系统，因此副本系统不应重复“自动获得收益”的体验，而应承担以下职责：

- 提供主动游玩的核心体验
- 提供路线选择与分支决策
- 提供文本叙事与世界观沉浸
- 提供比挂机更高波动的收益（支线、XP、稀有掉落）
- 通过探索过程而不是纯结算页面制造乐趣

### 1.2 生化危机副本的核心体验

“生化危机”副本的设计目标是：

- **战斗全自动**：玩家不手操每回合攻击/技能
- **路线由玩家控制**：玩家决定先去哪、搜哪里、是否冒险
- **文本驱动推进**：房间文本、日志、NPC 对话共同构成副本氛围
- **探索影响战斗**：前面的探索会改变后续战斗难度、资源、支援与结局
- **有 NPC 引导与支援**：降低新玩家迷失感，同时让分支更有反馈

一句话总结：

> 玩家负责战略级选择，系统负责战术级执行。

---

## 2. 现代 MUD 风格的具体含义

本项目中的“现代 MUD 风格”不是传统手打命令行，而是：

- 用 **房间/节点** 组织副本结构
- 用 **日志与场景文本** 表达推进过程
- 用 **行动按钮** 替代手动输入命令
- 用 **自动战斗日志** 替代手操回合战斗
- 用 **状态变量（flags）** 记录探索结果与分支后果

因此，副本体验应该类似：

1. 进入某个房间
2. 阅读当前场景描述
3. 选择行动或前往相邻区域
4. 触发检定、战斗、剧情或掉落
5. 在日志中看到后果
6. 继续探索直到通关或撤离

## 3. 副本基础信息

- **副本 ID**：`resident_evil_lab`
- **副本名**：生化危机：研究所突破
- **类型**：手动探索副本 / 现代 MUD
- **战斗模式**：全自动
- **推荐定位**：首个正式副本模板
- **推荐规模**：首版 10 个主要节点左右

### 3.1 主目标

玩家需要完成以下目标中的核心链路：

1. 获取门禁卡
2. 恢复 B 区供电
3. 打开安全门
4. 穿过 B 区
5. 抵达撤离平台并撤离

### 3.2 额外目标

- 搜索支线房间获取资源
- 救下支援 NPC
- 获得终局情报
- 决定是否带走危险样本

---

## 4. 核心状态变量（Flags）

这些状态变量是首版实现时建议保留的核心字段：

| 变量 | 含义 |
|---|---|
| `powerRestored` | 是否恢复 B 区供电 |
| `keycardObtained` | 是否获得一级门禁卡 |
| `alarmTriggered` | 是否触发全面警报 |
| `sampleTaken` | 是否带走危险样本 |
| `npcRescued` | 是否救下安保员雷克斯 |
| `armoryUnlocked` | 是否获得武器库权限 |
| `monitorViewed` | 是否查看过监控 |
| `doctorTrusted` | 是否信任林博士 |
| `finalSupportReady` | 终局是否可触发 NPC 支援 |
| `finalBossWeakened` | 是否提前掌握最终敌人弱点 |

这些 flags 会共同影响：

- 节点文案
- 路径开放情况
- 战斗强度
- 终局支援
- 最终奖励与结局表现

## 5. NPC 设计

### 5.1 林博士（引导型 NPC）

**定位**：通讯中的引导者  
**作用**：

- 提供副本目标说明
- 提示某些房间的价值
- 帮助新玩家理解当前阶段做什么
- 用于承接剧情信息与氛围塑造

**建议风格**：

- 紧张但理性
- 似乎知道更多内情
- 不一定完全无保留

**典型台词风格**：

- “先别去样本室，那里最早失控。”
- “你需要门禁卡和 B 区供电，少一个都过不去。”
- “如果你想活着离开，就别在主走廊停留太久。”

### 5.2 雷克斯（支援型 NPC）

**定位**：可救助的安保员  
**作用**：

- 提供武器库权限
- 提供终局一次援护
- 提供 B 区敌情信息
- 让玩家前期探索行为对后期战斗产生明确回报

**设计原则**：

- 不是必须救
- 救了会降低终局压力
- 不救不会卡关，但会损失支援与部分收益

## 6. 副本整体地图结构

```text
【实验室大门】
    ↓
【主走廊】
 ├─【值班室】
 ├─【样本室】
 ├─【配电间】
 └─【安全门】（需条件开启）
        ↓
     【B区通道】
     ├─【医务室】
     ├─【武器保管室】
     ├─【监控室】
     └─【撤离平台】
```

设计思路：

- `主走廊` 是前半段的中枢
- `B区通道` 是后半段的中枢
- 玩家会在“主线推进”和“支线探索”之间做选择
- 不是纯线性流程，而是“有限分支 + 明确主目标”结构

## 8. 节点详细设计

## RE_01【实验室大门】

**类型**：开场节点  
**作用**：建立生化氛围，引入林博士，告诉玩家“这是探索副本”。

**场景关键词**：

- 猩红警报灯
- 半掩的大门
- 消毒水与血腥味
- 门后的撞击声

**典型动作**：

- 进入设施
- 检查门口尸体
- 尝试呼叫幸存者

**主要收获**：

- 初始补给
- 开启林博士通讯
- 初步建立“谨慎探索”的氛围

---

## RE_02【主走廊】

**类型**：前半段 Hub 节点  
**作用**：提供路线选择，是前期的中枢房间。

**可前往分支**：

- 值班室
- 样本室
- 配电间
- 安全门

**动态变化建议**：

- 若 `alarmTriggered = true`：更频繁发生遭遇战
- 若 `powerRestored = true`：照明恢复，文本与提示变化

**设计意义**：

- 让玩家自己决定顺序
- 不把副本做成“下一页、下一页”的纯流程

---

## RE_03【值班室】

**类型**：低风险探索点  
**作用**：提供门禁卡、背景信息和基础补给。

**关键内容**：

- 守卫尸体上的门禁卡
- 值班日志
- 值班终端
- 储物柜补给

**典型动作**：

- 查看值班记录
- 拿取门禁卡
- 连接终端
- 搜查储物柜

**关键结果**：

- `keycardObtained = true`
- 获得 A 区背景信息
- 提示样本室和 B 区风险

---

## RE_04【样本室】

**类型**：高风险高收益房间  
**作用**：体现“贪收益要承担风险”的设计原则。

**氛围关键词**：

- 白色冷雾
- 破裂培养槽
- 拖痕
- 冷藏柜

**典型动作**：

- 搜索实验台
- 打开冷藏柜
- 提取危险样本
- 立即离开

**关键结果**：

- 可能获得稀有掉落
- 可能触发实验体战斗
- 可能获得终局弱点资料
- `sampleTaken = true`

**设计原则**：

- 玩家应当明确感受到“高收益 = 高风险”
- 样本是本副本最典型的“风险收益开关”

---

## RE_05【配电间】

**类型**：主线推进节点  
**作用**：控制主线推进节奏，改变研究所整体状态。

**典型动作**：

- 重启主电力
- 检查短路区域
- 强行合闸
- 离开

**关键结果**：

- `powerRestored = true`
- 可能 `alarmTriggered = true`

**设计意义**：

- 让玩家感觉自己在“改变设施状态”
- 不是单纯拿钥匙开门，而是推动整个副本世界变化

---

## RE_06【安全门】

**类型**：门槛节点  
**作用**：验证前期探索成果。

**通过条件**：

- `keycardObtained = true`
- `powerRestored = true`

**典型动作**：

- 刷卡开门
- 强行破门
- 返回继续探索

**关键结果**：

- 成功进入 B 区
- 或因强破触发高风险战斗/警报

**设计意义**：

- 让前面的探索具备明确回报
- 避免所有支线都变得“可有可无”

---

## RE_07【B区通道】

**类型**：后半段 Hub 节点  
**作用**：提升敌情密度，转入终局准备阶段。

**可前往分支**：

- 医务室
- 武器保管室
- 监控室
- 撤离平台

**特点**：

- 比 A 区更危险
- 自动战斗遭遇频率更高
- 氛围更偏“已经彻底失控”

---

## RE_08【医务室】

**类型**：NPC 支援节点  
**作用**：给玩家一个“救/不救”的中期选择，并反馈到终局支援。

**关键内容**：

- 重伤的安保员雷克斯
- 医疗物资
- 武器库权限芯片

**典型动作**：

- 救助雷克斯
- 询问 B 区情况
- 拿取医疗物资
- 放弃并离开

**关键结果**：

- `npcRescued = true`
- `armoryUnlocked = true`
- `finalSupportReady = true`

**设计意义**：

- 让玩家的探索行为对后续战斗产生明显影响
- 同时加入一点道德抉择味道

---

## RE_09【武器保管室】

**类型**：战斗增益房间  
**作用**：把探索结果转化为实打实的自动战斗优势。

**可进入条件**：

- `armoryUnlocked = true`
- 或监控室远程解锁

**典型动作**：

- 取用武器补给
- 拿取防护装备
- 查看安保记录

**关键结果**：

- 提升后续自动战斗表现
- 可进一步削弱终局战压力

---

## RE_10【监控室】

**类型**：信息控制节点  
**作用**：给玩家终局预判与设施控制能力。

**典型动作**：

- 查看撤离平台监控
- 远程解锁设施
- 下载设施日志
- 联系林博士确认路线

**关键结果**：

- `monitorViewed = true`
- 可获得终局敌情报
- 可远程解锁武器保管室等房间

**设计意义**：

- 让信息本身成为一种资源
- 增强“探索不只是找补给，还在找答案”的感觉

---

## RE_11【撤离平台】

**类型**：最终节点  
**作用**：最终战 + 路线结果汇总 + 结局触发。

**终局敌人建议**：

- `B区失控实验体`
- 或 `T-103试作融合体`

**终局难度由以下因素共同影响**：

- `sampleTaken`
- `alarmTriggered`
- `monitorViewed`
- `armoryUnlocked`
- `finalSupportReady`
- `finalBossWeakened`

**设计原则**：

- 终局不是完全独立的一战
- 而是玩家整趟探索结果的集中体现

---

## 9. 战斗插入原则

由于本副本的战斗模式是全自动，因此战斗不应该喧宾夺主，而应作为探索中的高张力事件插入。

### 9.1 玩家控制层

玩家控制的是：

- 去哪里
- 先搜哪里
- 是否冒险
- 是否救人
- 是否带走样本
- 是否准备更充分后再去终局

### 9.2 系统自动层

系统负责：

- 遭遇敌人
- 战斗结算
- 技能/攻击日志输出
- 伤害、掉落与战斗后果

### 9.3 推荐战斗点

**必定战斗**：

1. 样本室实验体
2. B 区通道中后段遭遇
3. 撤离平台最终战

**可选战斗**：

1. 值班室守卫尸变
2. 配电间遭遇战
3. 警报后的主走廊巡游敌人

设计重点：

- 战斗用于增强紧张感
- 不要把副本做成连续刷怪房间
- 保持“探索 > 战斗频率”的节奏关系

---

## 10. 推荐主线路径

推荐主线体验链路如下：

```text
实验室大门
→ 主走廊
→ 值班室（拿门禁卡）
→ 配电间（恢复供电）
→ 安全门（进入 B 区）
→ B区通道
→ 医务室（救雷克斯）
→ 监控室（看终局情报）
→ 武器保管室（拿增益）
→ 撤离平台（最终战）
```

这是“标准完整体验路线”。

---

## 11. 推荐高风险路线

高风险高收益路线如下：

```text
实验室大门
→ 主走廊
→ 样本室（拿危险样本）
→ 配电间（强行合闸）
→ 安全门（强闯）
→ B区通道
→ 跳过医务室/监控室
→ 撤离平台
```

这条路线应具有以下特点：

- 更高奖励
- 更差信息条件
- 更弱支援
- 更强终局压力

适合作为二刷或进阶路线。

---

## 12. 结局设计方向

首版建议至少做 3 种结局表现：

### 12.1 标准撤离

特征：

- 完成主线
- 未引发过高额外风险
- 正常击败最终敌人并离开

### 12.2 高风险高收益撤离

特征：

- 带走样本
- 承受更高终局压力
- 奖励更高
- 可作为后续剧情钩子

### 12.3 惨胜撤离

特征：

- 警报大量触发
- 缺少支援
- 战前准备不足
- 虽通关但表现更压抑

结局重点：

- 让玩家感受到“我走的不是固定流程”
- 让前面每个决定在终局有回音

## 13. AI/实现时的工程建议

### 13.1 数据结构建议

后续实现时建议将本副本定义为一个独立 TS 文件，例如：

- `src/data/scenarios/resident-evil.ts`

其中包含：

- 副本基础信息
- 起始节点
- 节点定义
- 动作定义
- 条件/flags
- 战斗触发定义
- 奖励定义

### 13.2 状态管理建议

副本运行态建议独立于玩家永久状态：

- 玩家永久状态：属性、货币、XP、支线、技能
- 副本运行态：当前节点、访问记录、flags、日志、临时战斗状态

### 13.3 实现优先级建议

**P0 必做节点**：

- `RE_01`
- `RE_02`
- `RE_03`
- `RE_05`
- `RE_06`
- `RE_07`
- `RE_11`

这些足以先跑通一个主线骨架。

**P1 强烈建议加入**：

- `RE_08` 医务室
- `RE_10` 监控室

这两者最能强化“现代 MUD”味道。

**P2 可后补**：

- `RE_04` 样本室
- `RE_09` 武器保管室

它们更偏向“高风险收益”和“二刷差异”。

## 14. 什么时候适合开始改代码

如果以下内容已经基本确认，就已经适合开始动代码：

1. 副本核心定位确定：自动战斗 + 玩家控路线
2. 节点结构确定：主走廊 / B 区通道双 Hub
3. 首版关键节点确定：至少 P0 这批节点不再大改
4. 关键 flags 已经确定
5. NPC 职责明确：林博士负责引导，雷克斯负责支援

**结论**：

> 现在就适合开始改代码，但建议先做“副本骨架”和“节点运行系统”，不要一开始就追求全文案完整或复杂战斗细节。

推荐第一阶段编码顺序：

1. 搭副本类型定义（TypeScript）
2. 搭副本运行 store
3. 把 `RE_01 ~ RE_07 ~ RE_11` 跑通
4. 接入日志系统
5. 再补医务室、监控室、样本室、武器库等支线

## 15. 最终设计结论

“生化危机：研究所突破”不应被做成：

- 一个自动刷怪页面
- 一个简单副本列表 + 继续按钮
- 一个只有结算没有过程的副本容器

它应该被做成：

> 一个以房间探索为骨架、以文本日志为表现、以自动战斗为事件、以 NPC 与状态变量为分支驱动的现代 MUD 式副本。

后续其他副本（如异形、咒怨、死神来了）都可以在这一模板上扩展。
