六、数据结构建议
请不要把“普通攻击”写死在 UI 里，而是抽象成统一的 CombatAbility。
建议新增或整理成类似结构：
type TargetType =
  | 'enemy'
  | 'ally'
  | 'self'
  | 'cell';

type AbilityShape =
  | 'single'
  | 'aoe-diamond';

interface CombatAbility {
  id: string;
  name: string;
  description?: string;

  // 技能从施法者自身向外能选多远
  range: number;

  // AOE半径，单体技能可以是 0
  area: number;

  // 技能目标类型
  targetType: TargetType;

  // single = 点单位
  // aoe-diamond = 点格子，然后影响范围内单位
  shape: AbilityShape;

  // 是否敌对技能
  hostile: boolean;
}

普通攻击也当成一个 ability：
const basicAttackAbility = {
  id: 'basic_attack',
  name: '普通攻击',
  range: unit.attackRange,
  area: 0,
  targetType: 'enemy',
  shape: 'single',
  hostile: true,
};
这样以后手枪、喷火器、医疗包、手雷、异能技能都可以接进同一套 UI。
七、Action 结构要求
UI 不要直接改血量、位置、状态。
UI 只负责生成 Action，然后交给战斗引擎验证和执行。
建议行动结构：
type CombatAction =
  | MoveAction
  | UseAbilityAction
  | WaitAction;

interface UseAbilityAction {
  type: 'use_ability';
  actorId: string;
  abilityId: string;

  // 单体技能用 targetUnitId
  targetUnitId?: string;

  // AOE / 投掷物 / 地面技能用 targetCell
  targetCell?: {
    x: number;
    y: number;
  };
}
执行前必须验证：
1. 当前是否轮到 actor 行动；
2. ability 是否存在；
3. ability 是否可用；
4. 目标是否在 range 内；
5. 单体技能是否点中了合法目标；
6. AOE 技能是否点中了合法格子；
7. 结算后是否写入日志。

九、范围计算规则
当前只需要曼哈顿距离。
function getManhattanDistance(a, b) {
  return Math.abs(a.x - b.x) + Math.abs(a.y - b.y);
}
技能射程：
distance(actor.position, targetCell) <= ability.range
AOE 范围：
distance(targetCell, affectedCell) <= ability.area
不要考虑障碍物、地形、视线遮挡。
