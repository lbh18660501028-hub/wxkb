# 个人空间系统设计

## 目标定位

个人空间不再只是一个信息页，而是队伍在主神空间中的后勤中枢。

它承担三类核心功能：

1. 队伍物资管理：随身背包、个人仓库、物品转移。
2. 小队训练调度：六大基础属性训练设施，与挂机训练系统联动。
3. 房间可视化表达：用室内背景、热点、设施节点，把“个人空间”做成可感知的场景。

长期目标是让个人空间从固定场景逐步演进为可装修、可扩建、可升级的沙盒式基地。

## 当前数据结构

文件：`src/types/game.ts`

### PersonalSpaceState

```ts
export interface PersonalSpaceState {
  roomType: 'indoor' | 'outdoor'
  roomStyle: string
  currentEnvironment: string
  facilityPlacements: FacilityPlacement[]
  storage: Record<string, number>
  trainingAssignments: Record<string, AttributeKey | null>
  trainingProgress: Record<string, Partial<Record<AttributeKey, number>>>
}
```

### FacilityPlacement

```ts
export interface FacilityPlacement {
  id: string
  type: string
  x: number
  y: number
  level: number
  skin?: string
}
```

当前 `x` 和 `y` 是百分比坐标。

例如：

```ts
{ id: 'hotspot_gym', type: 'gym', x: 25, y: 43, level: 1 }
```

这代表健身房热点显示在房间视图宽度 25%、高度 43% 的位置。

这套结构已经为后续拖拽装修预留好了入口。

## 后续演进路线

### 阶段 1：固定背景图 + 热点

当前已经完成基本结构。

后续可以继续补：

- 接入真实室内背景图。
- 每种房间风格使用不同图片。
- 热点图标改成更细致的设施图标。
- 热点 hover 时显示更强的终端信息。
- 点击设施后，在右侧显示更完整的状态面板。

这一阶段成本低，效果明显。

### 阶段 2：设施位置配置化

当前已经部分完成。

后续需要把 `placementMeta` 从页面中抽离到独立配置文件。

建议文件：

```text
src/data/personalSpace.ts
```

建议内容：

```ts
export const PERSONAL_SPACE_FACILITY_META = {
  gym: {
    kind: 'facility',
    icon: '...',
    label: '健身房',
    attribute: 'strength',
    desc: '训练肌肉强度',
  },
}
```

这样页面只负责渲染，数据配置集中管理。

### 阶段 3：设施升级

每个设施已经有 `level` 字段。

可以加入：

- 升级消耗奖励点。
- 升级消耗支线剧情。
- 升级后提高训练速度。
- 升级后开放更多训练槽位。
- 升级后改变热点外观或背景设施外观。

示例设计：

| 等级 | 效果 |
| --- | --- |
| Lv.1 | 基础训练速度 |
| Lv.2 | 训练速度 +20% |
| Lv.3 | 允许两人同时获得额外加成 |
| Lv.4 | 解锁高级训练事件 |
| Lv.5 | 解锁特殊属性突破 |

### 阶段 4：拖拽装修

这一阶段开始接近沙盒玩法。

基础需求：

- 设施可以拖动。
- 拖动后保存 `x`、`y` 坐标。
- 支持重置布局。
- 支持切换编辑模式。
- 编辑模式下显示网格辅助线。

推荐交互：

```text
普通模式：点击热点打开功能面板
编辑模式：拖动设施改变位置
```

实现建议：

- 不需要引入大型拖拽库。
- 可以先用 Pointer Events 自己实现拖动。
- 保存时更新 `personalSpace.facilityPlacements`。
- 坐标继续使用百分比，适配不同屏幕。

### 阶段 5：网格装修

如果希望更像基地建设或沙盒游戏，可以把个人空间切成网格。

推荐规则：

```text
房间宽 12 格，高 8 格
设施占用 1x1、2x1、2x2 等尺寸
设施不能重叠
部分设施要求靠墙或靠近能源节点
```

新增字段示例：

```ts
interface FacilityPlacement {
  id: string
  type: string
  gridX: number
  gridY: number
  width: number
  height: number
  rotation: 0 | 90 | 180 | 270
  level: number
  skin?: string
}
```

这一阶段适合加入：

- 建造模式。
- 设施占地。
- 设施相邻加成。
- 不同房间主题。
- 房间容量限制。

### 阶段 6：房间功能与玩法绑定

个人空间最终可以成为一个长期成长系统。

可绑定的系统：

- 仓库容量由仓库设施等级决定。
- 训练速度由对应设施等级决定。
- 小队休息恢复由生活设施决定。
- 新人招募概率或成本受居住区影响。
- 装备维修、制造、分解由工坊决定。
- 特殊剧情 NPC 需要对应房间设施才能入住。

## 注意事项

- 不建议马上做完整房间编辑器。
- 先做固定背景与设施成长，收益最高。
- 拖拽装修要等设施系统稳定后再做。
- 坐标使用百分比比像素更适合响应式布局。
- 后续真实图片应放入 `public/images/personal-space/`。
- 所有新增状态都要进入存档，否则刷新后会丢失。
