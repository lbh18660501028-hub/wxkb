# 游戏数值平衡规则

本规则在讨论数值平衡、属性调整、公式修改等内容时生效。

## 数值集中管理
**所有数值常量必须在 `src/config/combat.ts` 中定义**，禁止在业务代码中硬编码。

## 核心公式

### HP公式
```
HP = BASE_HP(100) + vitality × HP_MULTIPLIER(10) + geneLockTier × GENE_LOCK_HP_BONUS(10) + volumeBonus
```
体积HP加成：微型-20%, 小型-10%, 中型0%, 大型+10%, 巨型+20%

### MP公式
```
MP = BASE_MP(0) + intelligence × INTELLIGENCE_MP_MULTIPLIER(10)
```

### 意志力
```
意志力 = 精神力 + 智力
```

### XP公式
```
升级所需XP = LEVEL_UP_BASE(100) × LEVEL_UP_MULTIPLIER(1.15)^(level-1)
```

### 经济系统
- 挂机产出：只有奖励点（无支线）
- 副本产出：支线线索 + XP + 奖励点
- 支线互换：3:1经典比例（3个低级→1个高级）
- 最大离线挂机：5小时

## 属性效果
| 属性 | 效果 |
|------|------|
| 肌肉强度(STR) | +1 科技伤害/点 |
| 神经反应(REA) | +1 速度/点, +1 闪避/点 |
| 智力(INT) | +10 MP/点, +1% 暴击/点 |
| 细胞活力(VIT) | +10 HP/点, +0.5 HP回复/点 |
| 精神力(SPI) | +1 意志力/点, +2 魔幻伤害/点, +1 魔幻防御/点 |
| 免疫强度(IMM) | +1% 状态抗性/点, +1% 伤害减免/点, +1% 抗暴/点 |

## 属性上限
- 无血统：属性上限5
- 血统D级：上限6，1天赋点
- 血统C级：上限11，3天赋点
- 血统B级：上限16，9天赋点
- 血统A级：上限21，27天赋点
- 血统S级：上限36，100天赋点

## 装备价格
| 等级 | 价格(奖励点) | 倍率 |
|------|-------------|------|
| D | 100 | 1x |
| C | 500 | 5x |
| B | 2,000 | 20x |
| A | 10,000 | 100x |
| S | 50,000 | 500x |

## 属性升级费用阶梯
| 属性值区间 | 费用(奖励点) |
|-----------|-------------|
| 1-5 | 250 |
| 6-10 | 500 |
| 11-15 | 750 |
| 16-20 | 1,000 |
| 21+ | 1,500 |

## 基因锁
- 被动：每阶全属性+1
- 主动1阶：全属性+2, 暴击+10%, 闪避+10%, 消耗1意志/回合
- 主动5阶：全属性+10, 暴击+50%, 闪避+50%, 消耗5意志/回合
- 解锁基础概率：1%

## 调整数值时的注意事项
1. 只修改 `config/combat.ts` 中的对应常量
2. 注意连锁影响：改HP倍率会影响所有角色的HP上限
3. 经济平衡：挂机产出 vs 副本产出 vs 商店价格 要协调
4. 存档兼容：公式改了但旧存档的数值可能需要迁移
5. 修改后应运行游戏测试实际效果
